CF1286E Fedya the Potter Strikes Back 题解

First Post:
2023-05-02

Last Update:
2023-02-14

事实证明卡住的地方比我预想的要弱智一些，构思的时候已经想到了均摊，但对 border 的变化规律似乎缺乏更深的理解，也对最值相关缺乏研究，导致没有想到另一个均摊。

当然，这题也带来一个直觉，就是 border 相关的工程一般比较轻量级。

第一眼是 border 的四种求法中的 SAM，后来发现显然不用这么复杂，考虑 KMP 的过程即可。

注意到一个前缀的所有 border 是 KMP 失配树上的一条祖先链。想借此维护，但变化量看上去并不小，主要是 \(nxt[i]\) 变为 \(nxt[i - 1] + 1\) 的时候，可能接上了一条完全不同的祖先链，然后就卡在这儿了。

但事实上，我们直接考虑每个 border 串，在加入一个字符之后，它要么长度也加 \(1\)，要么就不是 border 串了。

注意到处理完上述变化之后，最多只会有一个长度为 \(1\) 的 border 串新加入进来（因为剩下的 border 串都是由上一轮的 border 整体加 \(1\) 而来）。

所以删除操作可以暴力执行，我们可以找到每个要删除的 border 串，然后更新相应的信息。

那么如何快速找到那些要删除的 border 串呢。

之前说过，一个前缀的所有 border 是失配树上的祖先链，也就是 \(nxt_i,nxt_{nxt_i},\dots,0\)，我们对于每个 \(i\)，维护最深的一个后继字符与它不同的祖先 \(anc[i]\)，即满足 $s[i + 1] != s[anc[i] + 1] $ 的最深的点。在遍历祖先的过程中，使用 \(anc[i]\) 加速即可。

至于 border 串长整体 \(+1\) 怎么做，设当前加入了 \(n\) 个字符，考察一个长度为 \(l\) 的 \(border\) 串，它的区间从加入前的 \([n - l + 1,n - 1]\) 变为了 \([n - l + 1,n]\)，我们发现，对于这些串，它们的权值只是对当前加入的 \(w_n\) 取了个 \(\min\)。

那么现在就是要维护一堆数，支持对所有数取 \(\min\) 和维护和。

我们可以用 map 维护每种权值的出现次数，要对 \(w_i\) 取 \(\min\) 时，把比 \(w_i\) 大的权值都直接变为 \(w_i\) 即可。

考察 map 中不同元素个数的变化情况，容易证明这里的均摊复杂度是 \(\Theta(n \log n)\) 的。

所以总时间复杂度是 \(\Theta(n \log n)\) 的。

这是一道总结 border 变化规律和最值变化规律的好题，对向后加入字符的问题，border 有比较简明的变化方式；对很多数不断地 \(\operatorname{chkmin}\)，也可以使用均摊复杂度做到优秀的结果。

思考的大方向没有什么问题，但是细节并没有想清楚，这是需要改进的。
