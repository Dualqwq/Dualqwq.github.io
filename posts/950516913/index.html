AGC007D

简单题，虽然对数轴上游走的题还是不太会。

一眼看出了状态，转移懒得编了（以后还是要编转移，现在编转移的能力有些弱）。

容易发现，路径一定是先往前走一段，再折回一段，再往前走，走了一段再折，再走。

说白了，一个点要么被经过一次，要么被经过三次。

那么设 \(f[i]\) 表示收到前 \(i\) 个糖果的最小时间，此时人一定在 \(x_i\)。

那么枚举位置 \(j\)，\(j + 1 \sim i\) 中的点被一次折返收完，那么有转移 \(f[i] = \min\{f[j] + (x[i] - x[j]) + \max(2(x[i]-x[j+1]),T)\}\)

写出这个转移式子之后，优化非常简单，分类讨论那个 \(\max\) 取到谁即可。因为 \(x\) 和 \(f\) 均不降，转移容易。

AGC007E

好题。

题意：给出一棵二叉树，每个节点只有 \(2\) 或 \(0\) 个儿子。

要求你从 \(1\) 走回 \(1\)，每天你可以从二叉树的一个叶子走到另一个叶子，或从 \(1\) 走到某个叶子，或从某个叶子走到 \(1\)。要求你经过二叉树上的所有叶子，且每条边被经过恰好两次。

你的花费是最大的，从叶子到叶子的那一天经过的边的权值和，求最小花费。

\(n \le 2 \times 10^5\)。

首先二分答案，然后设 \(bool\) 状态 \(f_{x,a,b}\) 表示从 \(x\) 开始游走，能否在第一天走过不超过 \(a\) 的距离，在第二天走过不超过 \(b\) 的距离。

那么有转移 \(f_{x,a,b} = OR_{k_1,k_2,k_1 + k_2 \le mid} f_{ls,a,k_1} \and f_{rs,k_2,b}\)

交换 \(ls,rs\) 后再转移一遍即可。

当时只是觉得，这个转移形式有点双指针，然后想着能不能只记录一维，觉得挺假就没往下想了。

事实证明，确实需要双指针。

首先，对于同为 \(true\) 的状态 \(f_{l_1,r_1}\) 和 \(f_{l_2,r_2}\) ，如果 \(l_1 \le l_2,r_1 \le r_2\)，那么后者完全无用。

所以你按 \(l\) 升序排序之后，\(r\) 一定是降序的。

显然，对于每一个 \(f_{ls,l_1,r_1}\) 都只需要考虑一个令 \(r_2\) 最小且满足转移式的 \(f_{rs,l_2,r_2}\)。所以，每一个 \(f_{ls}\) 中的状态都只需要与一个对应的 \(f_{rs}\) 转移到 \(f_x\) 中的唯一一个状态。所以，设左右子树的状态数分别为 \(x,y\)，那么一次转移只会增加最多 \(2 \times \min\{x,y\}\) 个状态（因为要把 \(ls,rs\) 交换后再转移一遍，所以要乘个 \(2\)）。

根据启发式合并的复杂度分析，总状态数是 \(\Theta(n \log n)\) 的。

找到每个状态对应的状态可以使用双指针，总时间复杂度是 \(\Theta(n \log n \log W)\) 的。

AGC007F

题意：

Shik 需要抄写一个字符串，一开始，纸面上有一个字符串 \(S_0\)，每次她写下字符串 \(S_i\) 的时候，\(S_i[j]\) 有可能与 \(S_{i-1}[j]\) 相同，也可能与 \(S_{i}[j-1]\) 相同。

Shik 希望抄写尽量少的字符串，就能使最后再纸上的字符串与 \(T\) 相同，具体地，她希望找到最小的整数 \(K\)，使得存在一种方案让 \(S_K = T\)。

\(|S_0| = |T|,1 \le |S_0| \le 10^6\)。

首先把图画出来，\(S_0\) 中的一个字符肯定对应的是 \(T\) 中的一段区间。每个字符相当于在平面上向右或向下走。

当时的直觉是从后往前做，每个字符的匹配段都往右靠一定不劣，然后根据每个字符对应的区间设计 DP。

事实上，这样设计 DP 反倒会丢失路径信息。

考虑我们在不改变每个字符所对应区间的情况下，把中间走过的路径尽量往右靠，那么一定不劣。

而具体的策略就是：一直尽量右移，直到走到要覆盖的左边界，然后一直向下，最后横向覆盖。

那么那些横坐标大于等于 \(j\) 的拐点要弹出。

然后，原来所有的拐点都会往左下方平移一格（因为新折线要尽量贴近原来的折线）。

最后，如果 \(i \ne j\) ，那么在横坐标为 \(i\) 的地方会有一个拐点。
