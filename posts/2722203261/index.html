CF1770G 题解（分治 NTT 在格路计数中的应用）

First Post:
2023-05-02

Last Update:
2023-03-02

之前没有接触过这方面的应用（没想到分治 NTT 这么强大），故记录一下。

首先我们考虑一个格路计数模型，即从 \((0,0)\) 出发，向右或向上走，要走到 \((n,m)\) ，但第 \(i\) 列只存在行数不大于 \(c_i\) 的点，\(c_i\) 单调不降，且 \(c_n = m\) ，求路径条数。

上述的网格是个阶梯状的图形，用分治 NTT 解决阶梯网格的格路计数是很经典的应用（虽然此前我从来没听说过）。

首先考虑 \(n^2\) DP，这是简单的，即 \(dp_{i,j} \to dp_{i+1,j},dp_{i,j} \to dp_{i,j+1}\)。

但是直接这么做很难优化，本质上是因为这个 DP 没有明显的阶段性，即同一个 \(i\) 内的所有 \(j\) 都会互相转移。

我们修改 DP 状态，第一维记录 \(x + y\)，第二维记录 \(y\) ，那么就会有 \(dp_{k,j} \to dp_{k + 1,j},dp_{k,j} \to dp_{k+1,j+1}\)。

为了避免在 \(k > n\) 的时候还会有 \(j \ge k - n\) 的限制，考虑从 \((n,0)\) 开始沿左上-右下方向把阶梯网格劈成两半，计算从 \((0,0)\) 和 \((n,m)\) 走到这条斜对角上的点 （\(i + j = n\)） 的方案数，容易发现这两个问题是对称的。

综合上述分析，我们将问题转化为，从 \((0,0)\) 出发，每次可以向右或向右上走一步，且在第 \(i\) 列只存在行数不大于 \(c_i\) 的点，走到最后一列每一个点的方案数。容易证明转化完后的 \(c_i\) 仍然是不降的，且满足一个优秀的性质：\(c_{x + 1} - c_x \le 1\)。

事实上，转化后的 \(c_i\) 就是 \(x + y = i\) 的点的个数，就算这个阶梯形网格是个 \(n \times m\) 的矩形，\(c_{i + 1} - c_i\) 也不超过 \(1\)。

经过上述的转化之后，问题终于有了明显的分层，可以考虑优化转移了。

考虑一个 \(dp_l\) 对 \(dp_r\) 的贡献，设 \(x = c_r - (r - l)\)，我们发现，对于 \(j \le x\)，\(dp_{l,j}\) 转移到 \(dp_r\) 时不会受到 \(c\) 的限制（因为 \(c_{i + 1} - c_i \le 1\)）。

（图源自 @Alex_Wei，对于 \(j \le x\) 的部分，它就算一路往上冲，也不会超过上图的红线。）

那么从 \(dp_{l,j_1}(j_1 \le x)\) 转移到 \(dp_{r,j_2}\) 的系数算起来就很容易了，就是 \(\binom{r - l}{j_2 - j_1}\)。

这是一个很明显的卷积形式，我们设 \(F_i = dp_{l,i},G_i = dp_{r,i},H_i = \binom{r - l}{i}\)，那么我们将 \(F_{0 \dots x}\) 与 \(H_{0 \dots r - l}\) 进行卷积， 将这一部分贡献加到 \(G_{0 \dots c_r}\) 上。

对于 \(j > x\) 的 \(dp_{l,j}\)，我们尝试将区间分为 \([l,mid]\) 和 \((mid,r]\) 递归处理，即先将左区间的贡献加到 \(dp_{mid}\) 上，再用 \(dp_{mid}\) 的一部分去更新 \(dp_r\)。

把这两部分的贡献加起来，所有 \(k \in [l,r)\) 的 \(dp_k\) 对 \(dp_r\) 的贡献就计算完毕了。

考虑实现这么一个算法流程：

设 \(Solve(l,r,\Delta,F)\) 表示当前还没确定 \(dp\) 值的区间为 \((l,r]\)，传入多项式的第 \(i\) 项表示 \(dp_{l,i + \Delta}\)，返回多项式的第 \(i\) 项表示 \(dp_{r,i + \Delta}\) 。这也可以看作是让所有 \(i \in [l,r]\) 的 \(c_i\) 全部减去 \(\Delta\)，传入的 \(F_i = dp_{l,i}\)，返回的 \(G_i = dp_{r,i}\)。下文将采用这种视角进行叙述。

设 \(x = c_r - (r - l)\)，对于 \(j \le x\)，将 \(F_{0 \dots x}\) 与 \(H_{0 \dots r - l}\) 卷积，将得到的结果加到 \(G_{0 \dots c_r}\) 上面。

对于 \(j > x\)，分治下放，设 \(F' = F_{x + 1 \dots c_l}\)，那么顺次执行 \(F' \gets Solve(l,mid,\Delta + x + 1,F'),F' \gets Solve(mid,r,\Delta + x + 1,F')\)。

将最后得到的 \(F'\) 也加到 \(G_{0 \dots c_r}\) 上面。然后返回 \(G\) 即可。

对于边界，\(r - l = 1\) 的处理是平凡的。

分析复杂度。对于下传的 \(F'\)，其长度显然不超过 \(c_r - x = r - l\)，也就是说，对于 \(Solve(l,r,\Delta,F)\)，\(F\) 的长度不会超过当前分治区间的父区间的长度，即 \(2(r - l)\)。设 \(n,m\) 同阶，则该算法复杂度与一般的分治 NTT 相同，为 \(\Theta(n \log ^2 n)\)。

终于可以说回原题了。

题意：给出一个括号串 \(S\) ，设 \(k\) 为最少需要删去的字符数使得原串是个合法括号串，求删去 \(k\) 个字符后原串合法的方案数，对 \(998244353\) 取模。

\(1 \le |S| \le 5 \times 10^5\)。

首先，我们要删去的字符一定形如 ))))(( ，即一段右括号再加上一段左括号。

我们考虑找到这个分界点。

把左括号看成 \(1\),右括号看成 \(-1\)。考察这个序列的前缀和。

我们一定一个位置为 \(\text{Special}\) 的，当且仅当它对应的数是前缀最小值。因为 \(\text{Special}\) 位置的数一定是在它之前的位置的数的最小值减 \(1\)，所以每遇上一个 \(\text{Special}\) 位置，我们就需要在这个位置或其之前删掉一个右括号。

那我们就将最后一个 \(\text{Special}\) 位置，当作上述的分界线，在这个位置左边，我们只删右括号；在这个位置右边，我们只删左括号。

这两个部分是本质相同的，下文以删右括号的部分为例。

那么我们对于每个前缀，选中的右括号数一定不小于 \(\text{Special}\) 位置的数量。

那么我们可以设 \(dp_{i,j} (j \ge 0)\) 表示前 \(i\) 个右括号 ，选中的右括号数减去 \(\text{Special}\) 位置的数量为 \(j\) 的方案数。

如果 \(i + 1\) 是个 \(\text{Special}\) 位置，转移到 \(dp_{i + 1,j/j - 1}\)，否则转移到 \(dp_{i + 1,j/j+1}\)。

这看上去就挺格路计数的，可以套用上面的做法。因为限制只有 \(j \ge 0\)，我们甚至不需要记录 \(\Delta\)。

当然，还是稍有变形。具体地，记录 \([l,r]\) 中 \(\text{Special}\) 位置的个数 \(cnt\)。那么对于 \(j \ge cnt\)，\(dp_{l - 1,j}\) 转移到 \(dp_{r,j + p - cnt}\) 就不受 \(j \ge 0\) 限制的影响，转移系数为 \(\binom{r - l + 1}{p}\) 。

对于 \(j < cnt\) 的部分，递归下放去做即可，下放的多项式长度不会超过 \(r - l + 1\)，故复杂度为 \(\Theta(n \log ^2 n)\)。

这道题加深了我对分治 NTT 的理解。
