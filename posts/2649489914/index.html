loj3668 Sleeping in Class

题意：给出一个序列 \(a_1,a_2,\dots,a_n\)，一次操作可以将两个相邻的数 \(x,y\) 合并为 \(x + y\)，也可以将数 \(t\) 分解为两个相邻的数 \(x,t-x\)，问最后使得序列中的所有数都等于 \(V\) 的最少操作次数。可能无解。\(Q\) 次询问，每次给出不同的 \(V\)。

\(1 \le n,Q \le 10^5\)。

设 \(s_i\) 表示 \(a_i\) 的前缀和。

那么 \(V \nmid s_n\) 则无解。

否则最优策略一定如下：

维护当前准备参与操作的数 \(p\) ，初始 \(p = 0\)。从左到右扫描到 \(i\)，把 \(p\) 与 \(a_i\) 合并，如果此时 \(p > V\)，就不停地从 \(p\) 中分出 \(V\) 来。

观察这个过程，发现 \(p = s_i \bmod V\) （特例，如果 \(s_i \bmod V = 0\)，\(p = V\)）

我们发现，如果没有 \(s_i \bmod V = 0 (i < n)\) 的位置，那么分裂要进行 \(\dfrac{s_n}{V} - 1\) 次，合并要进行 \(n - 1\) 次，而每出现一个这样的位置，可以少一次分裂，少一次合并。故答案为 \(\dfrac{s_n}{V} - 1 + (n - 1) + \sum_{i=1}^{n-1} [V \mid s_i]\)。

我当时转化出来的东西没有这么简洁明了，所以寄了。

考虑如何求出 \(\sum [V | s_i]\)，注意到 \(V | s_n\)，所以我们只关心所有的 \(\gcd(s_i,s_n)\) 。考察 \(V\) 和 \(D = \gcd(s_i,s_n)\) 的质因子分解。设其分别为 \(\prod p_i^{b_i},\prod p_i^{c_i}\) ，那么 \(D\) 对 \(V\) 有贡献当且仅当 \(\forall i,b_i \le c_i\)。我们发现这是一个高维后缀和的形式，而这事实上也是可以做的，因为 \(d(s_n)\) 在 \(s_n \le 10^{18}\) 时大约只有 \(10^5\) 级别，而维数也是 \(\omega(s_n)\) ，很小。这里高维后缀和的时间复杂度为 \(O(d(s_n)w(s_n)\log d(s_n))\) （这里的 \(\log\) 是使用 map 进行离散化的复杂度）。

现在的问题在于如何求出 \(s_n\) 的质因子分解了。做过 AGC003D 的同学应该熟悉这样一个方法：设值域为 \(L\)，先将不超过 \(L^{1/3}\) 的质因子试除掉。如果剩下的数小于 \(L^{2/3}\)，那么其一定是一个质数 ，否则其质因子就被试除掉了。如果大于 \(L^{2/3}\)，那么先前试除掉的部分总共就不超过 \(L^{1/3}\)，而剩余部分一定是至多两个质数相乘的形式。此时 \(d(s_n) \le 4 \times \max_{1 \le i \le L^{1/3}} d(i)\)。

此题中 \(L = 10^{18}\)。对于最后的那种情况，可能的 \(V\) 只有几千个，直接暴力即可。

loj3669 Phone Number

题意：https://loj.ac/p/3669

我当时先思考的前两个包，因为只有两键的状态转移较为简单，只需记录一个 \(f[i]\) 便可通过。后来想直接从 \(f[i]\) 转移到 \(f[i + 4]\) 。发现难以配出简单系数，转移跟你填的那些数关联性较强，然后就不会了。

事实上大力 DP，设状态 \((i,t_0,t_1,t_2,v_0,v_1,v_2,v_3)\) 。\(t_0,t_1,t_2\) 分别表示第 \(i,i - 1,i - 2\) 位你填的是什么字符，\(v_t\) 表示前缀 \([1,i - t]\) 能否根据题目规则匹配得上。转移枚举下一个字符填什么，比较简单。

但是直接做会有 \(2^4 \times 9^3\) 个状态，过不去的。

发现当 \(v_3 = 0\) 时，\(t_2\) 已经没有用了（因为 \(t_2\) 只会用来帮我们判断 \([i - 2,i + 1]\) 能否匹配，从而接上 \([1,i - 3]\)，但如果 \([1,i - 3]\) 本身就无法匹配，我们也不用做这个判断了），同理，当 \(v_3 = v_2 = 0\) 时，\(t_1\) 无用，当 \(v_3 = v_2 = v_1 = 0\) 时，\(t_0\) 也无用了。全为 \(0\) 就是不合法状态了。

根据这个进行裁剪，状态数好像只减少了不到一半。

还有一个优化，即若 \(v_3 = 1\)，那么对于 \(t_0,t_1,t_2\)，如果它们中有一个数不在 \([i - 2,i + 1]\) 中，或者它们再加一个数也拼不出正方形，那么 \(v_3\) 就没用了，我们可以将其设为 \(0\)。对于 \(v_2,v_1\) 也可以类似处理。

考虑分析状态数的上界。如果 \(v_3 = 1\)，那么 \(t\) 的方案就是在 \([i - 2,i + 1]\) 种选三个数的方案数，只有 \(24\) 种选法，\(v_{0,1,2}\) 不确定，一共 \(8 \times 24 = 192\) 状态。如果 \(v_3 = 0,v_2 = 1\)，\(t\) 就只有 \(t_0,t_1\) 有用，且在 \([i-1,i+2]\) 中选，一共 \(12\) 种，乘上 \(v_0,v_1\) 的选法，一共 \(48\) 种。如果 \(v_3 = v_2 = 0,v_1 = 1\)，就是 \(8\) 种状态。\(v_3 = v_2 = v_1 = 0\) 就 \(1\) 种状态。总状态数有个上界 \(192 + 48 +8 + 1 = 249\) 。

据说实际状态大约只有 \(50\) 多个。可以用 unordered_map 存状态来进行转移。
