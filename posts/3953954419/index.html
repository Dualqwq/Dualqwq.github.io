CF1575C 题解

文章发布时间:
2023-05-02

最后更新时间:
2023-02-28

赛时想着用卷积处理散块，但不知道如何避免重复统计的问题，赛后看到题解做法大受震撼，故记录一下。有时一个观察能帮你省掉很多东西。

我们先考虑固定右端点 \(r\) ，计算所有左端点的答案。

一个关键但显然的观察是，\(r\) 和 \(r + m\) 的答案是相同的，这意味着我们只需计算 \(1 \le l \le nm,1 \le r \le n\) 的答案，最后把答案乘 \(m\) 即可。

设 \(sum(p) = \sum_{i=1}^p b_i\) ，即 \(b\) 序列的前缀和。如果 \(nm\) 不大的话，我们大可以预处理出 \(cnt[i]\) 表示满足 \(sum(p) \bmod k = i\) 的 \(p\) 的个数。然后扫描 \(r\) ，每次先让 \(cnt[sum(r)]\) 减 \(1\)，再提取出 \(cnt[sum(r) + sum(n)]\) 作为 \(r\) 的答案，再将 \(cnt[sum(r) + sum(n)]\) 加 \(1\) 。容易发现，上述过程正确地统计了所有 \(l < r\) 和 \(l > r\) 的子段，可以手算验证。

我们发现，上述做法的瓶颈在于，如何快速地预处理 \(cnt\)。

如果 \(sum(n) \bmod k = 0\)，处理起来比较容易。

否则，我们希望对于每个 \(1 \le l \le n\)，我们能把 \(sum(l),sum(l + n),\dots sum(l + (m - 1)n)\) 的贡献一起加入到 \(cnt\) 中，即需要对于 \(t \in [0,m)\) ，令 \(cnt[(sum(l) + t \cdot sum(n)) \bmod k]\) 自增 \(1\)。事实上，考虑一张图，对于每个点 \(i\)，连边 \((i,(i + sum(n)) \bmod k)\) ，最后会形成一个长度为 \(k\) 的环（因为 \(k\) 是质数），那么我们的操作相当于对环上连续的一段进行区间加。维护差分数组即可做到 \(\Theta(n + k)\)。

事实上，对于 \(k\) 不是质数的情况，上述做法也是可行的，只不过可能会形成若干个环，我们要对每个环分开考虑。
