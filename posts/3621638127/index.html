最近开的题都有些 atcoder,不过相比于看题解都看得一头雾水的 ARC127E,这题反而更加良心。

已经想到了构造可行解的方法，但没有抽象出判断可行解的条件。看来卡在比较弱智的地方确实是个比较常见的事情。希望某天可以踢出这临门一脚。

说到底还是在这方面没什么脑子（

题意：请构造序列 \(x\)，满足元素 \(i\) ($ 1 i n$) 在该序列中恰好出现 \(a_i\) 次，且任意连续 \(k\) 个元素都互不相同，或报告无解。有解则输出字典序最小的解。

\(1 \le k \le n \le 500,1 \le \sum a_i \le 200000\)。

首先考虑如何判断有无解，尝试设计一个贪心。

既然是贪心，那就先满足最紧的限制，即先考虑 \(a_i\) 较大的元素，那么有一个方法就是隔 \(k\) 个位置放一个。

然后容易想到对 \(x\) 序列每 \(k\) 个分一块，那么最先考虑的元素当然是放在每一块的起始位置。设块数为 \(c\)，最后一块长度为 \(g\)（因为最后一块长度最小）。

那么显然 \(a_i \le c\)，且 \(\sum [a_i = c] \le g\)。（就是这一步没有抽象出来）。

构造的方法是简单的，按 \(a_i\) 从大到小考虑每个元素，然后每次选择空位最多的段插进去即可。

然后基于此设计贪心使得字典序最小。

然后可以猜出一个贪心，即每次找到前 \(k - 1\) 个位置中未出现的最小的元素，如果此时 \(\sum [a_i = c] = g\)，那就只能选 \(a_i = c\) 的元素，然后处理后面长度 \(-1\) 的子问题。

证明正确性，我们就是要证明，只要初始序列有解，每个时刻就都能选一个数出来。

反证法，考察第一个不合法的时刻。

  1. 如果 \(\sum [a_i = c] = g\)，因为无数可选，所以 \(a_i = c\) 的所有数都在前面 \(k - 1\) 个位置出现过，而如果将前面 \(k - 1\) 个数加入进来重新分块，那么此时 \(g' \gets g - 1\)，但仍然有 \(g\) 个 \(a_i = c\) 的数，那么会有 \(\sum [a'_i = c'] > g'\)，但在合法条件下，这个策略肯定不会让这种事发生，故矛盾。
  2. 如果 \(\sum [a_i = c] < g\)，此时选数没有限制，无数可选只可能是因为 \(a_i > 0\) 的数只有至多 \(k - 1\) 个，设当前位置加上后面有 \(S\) 个空位，如果 \(S \ge k\)，那么显然后面无解，而 \(\sum [a_i = c] < g\) 保证了后面有解，故矛盾。所以 \(S \le k - 1\)，那么此时 \(c = 1,g = S\)，所有的 \(a_i\) 都会等于 \(c\)，所以 \(\sum [a_i = c] = g\)，矛盾。

综上所述，上述贪心一定可以构造出解，时间复杂度 \(\Theta(n \sum a_i)\)。
